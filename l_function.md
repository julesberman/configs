# Goal

A single function `l` that replaces `ls` with an interactive picker for SSH/ML workflows.

# Requirements

- Inline (50% height, doesn't take over terminal)
- Arrow keys + fuzzy search via `fzf`
- Pretty: colored entries, rounded border, custom prompt/pointer/marker, Tokyo Night palette
- Directories sorted first; `.` and `..` pinned at top
- `.` = no-op, `..` = up one
- `→` enters dir, `←` goes to parent (nnn-style navigation inside the picker)
- Working preview pane on the right for every entry, including `.` and `..`
- Smart Enter dispatch:
  - dir → `cd` in
  - image / PDF → `timg` (inline render on Kitty/iTerm2/WezTerm/Sixel terminals)
  - CSV / TSV → `column` + `less -S`
  - JSON → `jq -C` + `less -R`
  - Markdown → `glow`
  - binary ML files (`.npy`, `.pt`, `.ckpt`, `.safetensors`, …) → just print name + size
  - archives (`.zip`, `.tar.*`) → list contents
  - everything else → `$EDITOR`
- Concise comments + install line at the top
- Works in **both bash and zsh** with one implementation

# Broader context

This sat inside a larger workflow question: viewing matplotlib plots / GIFs generated by Claude Code over SSH on a remote ML box. Three-tier summary we landed on:

1. **Canonical workflow:** modern terminal with inline-image protocol (iTerm2 / WezTerm / Kitty / Ghostty) locally + `timg` remotely. `plt.savefig(...)` → `timg foo.png` renders at full resolution in your local terminal.
2. **Claude Code integration:** Claude Code cannot render images inline in its CLI today (open feature request [#2266](https://github.com/anthropics/claude-code/issues/2266)). It can `Read` images and reason about them. Running `!timg foo.png` inside Claude Code won't render because escape sequences get swallowed.
3. **Community tools** (clipssh, claude-ssh-image-skill, cc-clip) all solve the *reverse* direction — pasting local clipboard images *into* remote Claude Code — not viewing remote plots locally.

# Key design decisions / lessons from bugs hit along the way

- **Don't try to maintain navigation state inside fzf via temp files + reload bindings** — fragile, racy. Use a `while` loop in the outer shell; each iteration is a fresh fzf invocation with `$cur` baked in.
- **Use `--expect=right,left`** to detect arrow-key navigation vs Enter without closing the picker permanently.
- **Embed absolute paths in the listing itself** (column 1) instead of trying to pass `$cwd` to the preview via environment — fzf's `{}` substitution + subshell env propagation is a quoting minefield.
- **Use a two-column TSV listing**: col 1 = clean absolute path (for preview + outer shell), col 2 = ANSI-colored display name. `--ansi --delimiter=$'\t' --with-nth=2` renders colors but `cut -f1/-f2` of fzf's output gives clean strings.
- **Cross-shell:** `export -f` is bash-only. Solution: embed the preview helper as a POSIX shell string and invoke via `sh -c '<script>' _ {1}` in `--preview`.
- **macOS caveat:** BSD `ls` lacks `--group-directories-first` and `--color=always`. Don't just call `ls` and hope — *also* don't hardcode `gls`. The function detects `gls` once (`command -v gls && _ls=gls`) and uses `$_ls` everywhere. The embedded preview script re-detects inside its `sh -c` subshell because `$_ls` isn't exported through fzf.
- **zsh `local path` is a tied-array footgun.** Lowercase `path` is bound to `PATH` via `typeset -T`. Declaring `local path` inside a function blanks PATH for the function and every subsequent external command dies with "command not found" — including `fzf`, which makes the error look like a missing dependency rather than a shadowing bug. Use `target` / `entry_path` instead. Same hazard applies to `manpath`, `fpath`, `cdpath`.
- **zsh `local x` (no `=value`) prints x's existing value** unless `TYPESET_SILENT` is set, and it's *unset* by default. Inside a `while` loop, the first iteration is silent (locals don't exist yet) but every subsequent iteration dumps `name=$'value'` lines to the terminal that look exactly like `typeset -p` output — except nothing in the function explicitly calls `typeset -p`. Fix: `[ -n "$ZSH_VERSION" ] && setopt local_options typeset_silent` at the top of the function. `LOCAL_OPTIONS` makes the change auto-revert on function exit.

# Final function

Lives in `~/.bashrc` and/or `~/.zshrc`. Tested in both shells (listing build, preview script via `sh -c`, filenames with spaces, Esc exit code propagation — all identical across bash 5.2 and zsh 5.9).
